# Cache Sync Webhook — Deprecation Spec

## Purpose

The webhook-based cache-sync capability (`cache-sync-webhook`) is DEPRECATED and REMOVED. It is fully replaced by `cache-sync-realtime` (Supabase Realtime CDC).

## REMOVED Requirements

### Requirement: HMAC signature verification

(Reason: Webhook server is eliminated; Realtime CDC uses outbound WebSocket with no inbound HTTP exposure.)
(Migration: Replaced by Supabase Realtime subscription authentication via existing `SUPABASE_URL` + `SUPABASE_KEY`.)

### Requirement: Payload validation

(Reason: No webhook endpoint exists; CDC payloads are server-generated by Supabase.)
(Migration: CDC payload structure (`record`, `old_record`, `type`, `table`, `schema`) replaces custom JSON payloads.)

### Requirement: Idempotent invalidation

(Reason: No webhook delivery to duplicate.)
(Migration: CDC events are inherently idempotent; `invalidate_guild()` remains idempotent.)

### Requirement: Cache invalidation effect

(Reason: Invalidation trigger changes from webhook POST to CDC event.)
(Migration: `cache.invalidate_guild()` is still called — same effect, different trigger in `cache-sync-realtime`.)

### Requirement: Server lifecycle

(Reason: aiohttp webhook server is deleted.)
(Migration: Realtime subscriber lifecycle replaces server lifecycle — starts in `setup_hook`, stops on `cog_unload`.)

### Requirement: Environment configuration

(Reason: Webhook env vars are no longer needed.)
(Migration: Remove `WEBHOOK_SECRET`, `WEBHOOK_HOST`, `WEBHOOK_PORT` from bot env; remove `WEBHOOK_URL`, `WEBHOOK_SECRET` from dashboard env. Realtime uses existing Supabase credentials.)
